背景

专业媒体资源，包括电视与电影的音视频制作，正日益迁移到云端存储。

随之增长的还有开发对应 Web 应用的兴趣，从而允许终端用户去操作这些资源，如编辑、查看质量、版本控制、时序文本创作等。虽然现在的 Web 平台已经进化支持自定义媒体应用，但专业的媒体制作应用还需要额外的能力，包括精确的时间轴、高保真的时序文本、高效的媒体处理方案、宽色域和高动态色彩范围等。

本次研讨会的目标？

本次研讨会旨在沟通 Web 平台社区与专业媒体制作社区，并探索 Web 平台技术变革以满足专业媒体制作的需求，包括：

  • 使用 Web 技术，如 WebCodecs, WebGPU, 和 Web Audio 进行专业媒体制作
  • 了解专业媒体制作工作流程的独特性能和格式要求
  • 界定现有的 Web 平台技术缺失与标准化机会

本次研讨会涵盖在 Web 平台进行的专业媒体制作，包括编辑、质量控制、色彩分级修正、样片、视觉特效、声音、母版制作、翻译与服务。基于云的处理与不使用 Web 平台的桌面应用并不在讨论范围里。类似地，不对内容进行操作的应用，如文件分享应用并不属于讨论范围。

会涵盖哪些话题？

我们欢迎有关 Web 和媒体制作之间交叉主题的提案。到目前为止，以下的建议话题已被接收。请 在 GitHub 提交 pull request 或 发布 issue 来提议更多的研讨会主题。您也可以邮件联系 François Daoust <fd@w3.org>.

  • 专业媒体制作相关的用例，例如： 本地化, 无障碍 essence 再创作, 质量监测 (QC), 制作, 量产, 等等。
  • 支持专业媒体制作的技术:
    • 视频编解码, 例如， AVC-I, JPEG 2000.
    • 沉浸式音效；
    • 时序文本，例如， IMSC.
    • 无障碍，例如，手语视频.
    • 高动态 (HDR) 与 宽色域 (WCG) 范围图像。请参照 Web 宽色域与高动态范围技术研讨会 报告里的详细信息。
  • 将现有/即将推出的 Web 技术应用于专业媒体制作场景：WebGPU、WebGL、WebAssembly、WebCodecs、WebRTC Encoded Media、Web Audio 等。
  • 使用 Web 应用进行客户端媒体编辑与使用 Web 应用进行远程桌面访问。
  • 精确的 essence 同步、视频的帧精度和音频的采样精度。
  • 通过基于 Web 的接口，对存储在云中的媒体信息、以及实时媒体制作工作流程进行操作时的延迟管理。
  • 访问客户端功能，例如显示能力。
  • 视觉特效 的实时应用。
  • 内容安全，包括使用细粒度的访问控制、内容加密和取证标记，以跟踪用户与产品内容的交互并允许泄漏归因 (attribution of leaks)。
  • 内容生产工作流程中的元数据创建和管理，包括有关内容来源和真实性的元数据。
  • 意识到如上所述的跟踪、归属和真实性证明会造成隐私危害，尤其是在专业制作用例之外，讨论限制它们在专业环境之外使用的机制，在使用时明确说明，并使它们选择实施。
  • 与过时系统和标准的兼容性，例如，timecode.
  • 实时的和打包的媒体制作工作流程的需求。
  • 使用 Web 技术的音频和视频贡献者，例如 WebRTC 或 WebTransport。

在媒体制作场景中，术语 essence 代表组成媒体信息的可感知的数据，包括图片、声音、时序文本和可以直接感知的其他形式的数据。

什么是 W3C？

W3C 是一个世界范围内开放的标准组织，汇集众多公司与社区成员共同围绕现有和新兴技术展开富有成效的讨论，其为 Web 标准设立了免版税的专利框架。W3C 主要关注客户端（浏览器）技术，并且拥有成熟的词汇表（或“本体”）开发历史。W3C 根据成员和社区输出的优先级开展相应标准化工作。

什么是 SMPTE?

SMPTE 是一个由媒体专业人士、技术人员和工程师组成的全球协会，旨在 合作推动所有和电影、电视和数字媒体产业相关的技术的进步。SMPTE 也是一个国际标准组织，产出标准、推荐实现和工程指南，为全球数十亿观众的动态图像内容服务。
